# Content Formats

Use these formats to keep content outputs practical and reusable.

## README Intro

Keep it to:

- what it is
- who it is for
- why it matters

Usually 2 to 4 sentences.

## Launch Blurb

Use 1 to 2 sentences.

Good for:

- GitHub description
- product listings
- preview cards

## Tweet Or Short Post

Keep it tight:

- hook
- what it does
- why it is cool or useful
- call to action if needed

## Changelog Summary

Lead with the user-visible change, not the refactor.

Good pattern:

- added
- improved
- fixed

## Thread Or Longer Release Post

Use only when the project has enough substance to justify it.

Sequence:

- problem
- solution
- standout details
- how to try it
